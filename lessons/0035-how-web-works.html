Lesson 0035: How the Web Works

Alamin kung paano gumagana ang web — HTTP requests, DNS, hosting, at ang pagkakaiba ng frontend at backend.

Lesson Goal Sa pagtatapos ng araling ito, ikaw ay makakapag-explain kung paano nagke-communicate ang browser at server, at makikilala ang iba't ibang HTTP methods at status codes.

HTTP/HTTPS at ang Request-Response Cycle

Kapag nag-type ka ng URL sa browser, nagse-send ang browser ng HTTP request sa server. Ang server ay nagre-respond ng HTML, CSS, JS, at iba pang resources. Ang HTTPS ay secure version ng HTTP na naka-encrypt ang data.

"Kapag nag-type ka ng URL, nagse-send ng request si browser, si server nagre-respond ng HTML na nire-render ng browser."

HTTP Methods

  • GET — Kumukuha ng data mula sa server (hal. mag-load ng webpage)
  • POST — Nagse-send ng data sa server (hal. mag-submit ng form)
  • PUT — Mag-update ng existing data
  • DELETE — Mag-delete ng data

HTTP Status Codes

200 OK          — Successful request
301 Moved       — Na-redirect ang URL
404 Not Found   — Hindi mahanap ang page
500 Server Error — May problema sa server

Domain Names at DNS

Ang domain name (hal. google.com) ay madaling tandaan na address. Ang DNS (Domain Name System) ang nagta-translate ng domain name papunta sa IP address ng server. Parang phonebook ng internet.

Frontend vs Backend

  • Frontend — Ang nakikita at ini-interact ng user. HTML, CSS, JavaScript ang gamit.
  • Backend — Ang nasa server. Nagpro-process ng data, kumokonekta sa database. PHP, Python, Node.js ang gamit.

2. Ano ang pinagkaiba ng GET at POST?
3. Ano ang ibig sabihin ng 404 status code?

Mini-Project: DevTools Network Tab

Buksan ang Chrome DevTools (F12) at pumunta sa Network tab. I-reload ang isang website at obserbahan ang mga requests: method, status code, at response time. Mag-screen capture o magsulat ng observation tungkol sa nakita mo.

Summary

  • Ang HTTP ay protocol para sa communication ng browser at server.
  • Ang GET ay for retrieving data, POST ay for sending data.
  • Ang DNS ang nagta-translate ng domain name sa IP address.
