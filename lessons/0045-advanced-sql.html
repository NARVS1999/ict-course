Lesson 0045: Advanced SQL — Joins & Subqueries

Pagsasama ng maraming tables gamit ang JOIN at pag-embed ng queries gamit ang subqueries.

Lesson Goal Matutunan ang INNER JOIN, LEFT JOIN, RIGHT JOIN, subqueries, at GROUP BY na may HAVING.

INNER JOIN

Ang INNER JOIN ay nagbabalik lamang ng mga rows na may tugma sa parehong tables. Halimbawa, kung gusto mong malaman kung sino ang um-order at anong produkto, pagdugtungin ang Customers at Orders tables.

SELECT c.customer_name, o.order_date, o.total
FROM Customers c
INNER JOIN Orders o ON c.customer_id = o.customer_id;

LEFT JOIN at RIGHT JOIN

Ang LEFT JOIN ay nagbabalik ng lahat ng rows mula sa left table, kahit walang tugma sa right table (magiging NULL ang right side). Ang RIGHT JOIN naman ay baligtad — lahat ng rows mula sa right table.

-- Lahat ng customers, kahit walang order
SELECT c.customer_name, o.order_id
FROM Customers c
LEFT JOIN Orders o ON c.customer_id = o.customer_id;

-- Lahat ng orders, kahit walang customer (rare)
SELECT c.customer_name, o.order_id
FROM Customers c
RIGHT JOIN Orders o ON c.customer_id = o.customer_id;

Table Aliases at Self-Join

Gamit ang alias (c, o), pinaikli ang pagsulat ng query. Ang self-join naman ay kapag ang isang table ay ni-join sa sarili nito — halimbawa, empleyado at manager na nasa iisang table.

-- Self-join: employees at kanilang managers
SELECT e.name AS employee, m.name AS manager
FROM Employees e
LEFT JOIN Employees m ON e.manager_id = m.employee_id;

GROUP BY at HAVING

Ang GROUP BY ay nag-grupo ng rows batay sa column. Ginagamit ito kasama ng aggregate functions. Ang HAVING naman ay parang WHERE para sa grouped data — nagfa-filter ng groups.

SELECT c.customer_name, COUNT(o.order_id) AS total_orders, SUM(o.total) AS total_spent
FROM Customers c
INNER JOIN Orders o ON c.customer_id = o.customer_id
GROUP BY c.customer_name
HAVING COUNT(o.order_id) >= 3
ORDER BY total_spent DESC;
"Kapag gusto mo pagsamahin ang data mula sa dalawang table, gumamit ng JOIN — i-link gamit ang foreign key."

Subqueries (Nested SELECT)

Ang subquery ay isang SELECT na naka-embed sa loob ng isa pang query. Pwedeng gamitin sa WHERE, FROM, o SELECT clause. Mas nababasa minsan ang subquery kaysa sa JOIN, ngunit maaaring mas mabagal.

-- Hanapin ang customers na may order na lampas sa average
SELECT customer_name
FROM Customers
WHERE customer_id IN (
    SELECT customer_id
    FROM Orders
    WHERE total > (SELECT AVG(total) FROM Orders)
);

Mini-Project: Orders & Customers Query

Sumulat ng SQL query na gagamit ng INNER JOIN para pagsamahin ang Orders at Customers tables. I-display ang customer name, order date, at total amount. Pagkatapos, gumamit ng GROUP BY at HAVING para ipakita lamang ang customers na may higit sa 2 orders.

Summary

  • INNER JOIN = tugma lang; LEFT JOIN = lahat mula left; RIGHT JOIN = lahat mula right.
  • Ang subquery ay SELECT sa loob ng SELECT; pwedeng gamitin sa WHERE, FROM, o SELECT.
  • GROUP BY ay nag-grupo ng rows; HAVING ay nagfa-filter ng groups (hindi tulad ng WHERE).
