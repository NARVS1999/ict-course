Lesson 0058: Virtualization & Containers

Pag-unawa sa virtualization — virtual machines (VirtualBox/VMware) at containers (Docker) — at ang pagkakaiba ng dalawa.

Lesson Goal Sa pagtatapos ng araling ito, ikaw ay makapagpapaliwanag ng virtualization at containers, makapag-differentiate ng VMs at containers, at makapag-run ng Docker "hello-world" container.

Ano ang Virtualization?

Ang virtualization ay ang paggawa ng virtual version ng isang resource — tulad ng server, OS, storage, o network. Sa pamamagitan ng hypervisor (tulad ng VirtualBox, VMware, o Hyper-V), pwedeng magpatakbo ng maraming virtual machines (VMs) sa iisang physical server.

Halimbawa: Sa isang laptop na Windows, pwedeng magpatakbo ng Ubuntu Linux sa loob ng VirtualBox. Parang computer sa loob ng computer.

"Ang VM ay parang isang buong computer na ginaya lang — may sariling OS, kernel, at apps. Pero magaan lang dahil software lang siya."

Ano ang Containers?

Ang container ay isang lightweight, standalone executable package na may kasamang lahat ng kailangan para tumakbo ang isang software — code, runtime, system tools, libraries, at settings. Ang pinakasikat na container platform ay Docker.

Ang container ay nagse-share ng OS kernel ng host machine, kaya mas magaan at mas mabilis compared sa VM. Hindi na kailangan ng separate OS para sa bawat container.

"Container parang kapsula na may lahat ng kailangan ng application — consistent sa kahit anong environment."

Docker Basics

  • Docker Image — Template o blueprint para sa container. Halimbawa: node:18, ubuntu:22.04, nginx:latest.
  • Docker Container — Running instance ng isang image.
  • Dockerfile — Text file na may instructions kung paano i-build ang Docker image.

Example Dockerfile

# Simple Dockerfile para sa Node.js app
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 3000
CMD ["node", "app.js"]

Summary

  • Ang virtualization ay paggawa ng virtual version ng computer (VM) gamit ang hypervisor.
  • Ang Docker ay ang pinakasikat na container platform.
  • Ang container ay mas magaan, mas mabilis, at nagse-share ng host OS kernel.
  • Ang Dockerfile ay ginagamit para i-build ang Docker images.
