Lesson 0047: Working with Databases in Backend Code

Pagkokonek ng database sa backend gamit ang PHP PDO, prepared statements, at transaction handling.

Lesson Goal Matutunan kung paano kumonekta sa database mula sa backend code, gumamit ng prepared statements, at mag-handle ng transactions.

Connecting to MySQL with PHP PDO

Ang PDO (PHP Data Objects) ay isang extension sa PHP para sa pag-access ng databases. Ito ay database-agnostic — pwedeng MySQL, PostgreSQL, o SQLite gamit ang iisang interface.

<?php
// Konek sa MySQL gamit ang PDO
$host = 'localhost';
$dbname = 'school_db';
$username = 'root';
$password = '';

try {
    $pdo = new PDO("mysql:host=$host;dbname=$dbname", $username, $password);
    $pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
    echo "Connected successfully";
} catch (PDOException $e) {
    echo "Connection failed: " . $e->getMessage();
}
?>

Prepared Statements (Proteksyon sa SQL Injection)

Ang prepared statement ay naghihiwalay ng SQL logic mula sa data. Sa halip na i-concatenate ang user input sa query, gumamit ng placeholders (?) o named parameters (:name). Ito ang pinakamabisang depensa laban sa SQL injection.

<?php
// Prepared statement gamit ang named placeholders
$stmt = $pdo->prepare("SELECT * FROM Students WHERE email = :email AND course = :course");
$stmt->execute([
    ':email' => $_POST['email'],
    ':course' => $_POST['course']
]);
$results = $stmt->fetchAll(PDO::FETCH_ASSOC);

// INSERT gamit ang prepared statement
$stmt = $pdo->prepare("INSERT INTO Students (first_name, last_name, email) VALUES (?, ?, ?)");
$stmt->execute(['Maria', 'Santos', 'msantos@email.com']);
?>
"Never concatenate user input sa SQL query — laging gumamit ng prepared statements."

Handling Transactions

Ang transaction ay grupo ng SQL operations na lahat ay dapat magtagumpay o lahat ay mabibigo (rollback). Ito ay importante para sa consistency — halimbawa, kapag nag-transfer ng pera: bawas sa sender, dagdag sa receiver. Kung bumagsak ang isa, i-rollback lahat.

<?php
$pdo->beginTransaction();
try {
    $pdo->exec("UPDATE Accounts SET balance = balance - 500 WHERE id = 1");
    $pdo->exec("UPDATE Accounts SET balance = balance + 500 WHERE id = 2");
    $pdo->commit();
    echo "Transfer successful";
} catch (Exception $e) {
    $pdo->rollBack();
    echo "Transfer failed: " . $e->getMessage();
}
?>

Pagination at Search Queries

Para sa malalaking datasets, gumamit ng pagination (LIMIT at OFFSET) at search (LIKE) para hindi ma-overwhelm ang user at server.

<?php
$page = isset($_GET['page']) ? (int)$_GET['page'] : 1;
$limit = 10;
$offset = ($page - 1) * $limit;

$search = '%' . $_GET['q'] . '%';
$stmt = $pdo->prepare("SELECT * FROM Students WHERE name LIKE :search LIMIT :limit OFFSET :offset");
$stmt->bindValue(':search', $search, PDO::PARAM_STR);
$stmt->bindValue(':limit', $limit, PDO::PARAM_INT);
$stmt->bindValue(':offset', $offset, PDO::PARAM_INT);
$stmt->execute();
?>

Mini-Project: PHP PDO Database Script

Sumulat ng PHP script gamit ang PDO na: (1) kumokonek sa MySQL database; (2) gumagamit ng prepared statement para kumuha ng records mula sa Students table; (3) nagdi-display ng results sa HTML table. Siguraduhing walang SQL injection vulnerability.

Summary

  • Gamitin ang PDO para sa database connection — mas secure at database-agnostic.
  • Prepared statements = proteksyon sa SQL injection; huwag mag-concatenate ng user input.
  • Transactions = consistency: lahat magtagumpay o lahat mabibigo (commit/rollback).
