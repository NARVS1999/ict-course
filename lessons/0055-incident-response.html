Lesson 0055: Incident Response & Ethical Hacking Intro

Pag-unawa sa proseso ng pagtugon sa security incidents at panimula sa penetration testing at ethical hacking.

Lesson Goal Matutunan ang incident response steps, ang konsepto ng penetration testing, at mga tools na ginagamit ng ethical hackers.

Incident Response Steps

Ang incident response ay ang proseso ng pag-handle ng security breach o attack. Ang mga hakbang:

  • Preparation — Maghanda ng plan, tools, at team bago mangyari ang incident.
  • Detection — Tukuyin na may incident na nangyayari (alerts, logs, user reports).
  • Containment — Pigilan ang pagkalat ng damage (i-isolate ang affected system).
  • Eradication — Tanggalin ang root cause (alisin ang malware, i-patch ang vulnerability).
  • Recovery — Ibalik sa normal ang operations (restore from backup, i-monitor).
  • Lessons Learned — Pag-aralan ang incident at magpatupad ng improvements.

Penetration Testing (Pen Testing)

Ang penetration testing ay simulated cyber attack laban sa isang system para suriin ang seguridad. Layunin nito na mahanap ang vulnerabilities bago pa man sila matuklasan ng mga malicious attackers. May tatlong approach: black-box (walang prior knowledge), white-box (full knowledge), at grey-box (partial knowledge).

Tools: Kali Linux, Wireshark, Nmap

Kali Linux — Isang Linux distribution na may pre-installed na security tools para sa penetration testing at ethical hacking. Wireshark — Network protocol analyzer na kumukuha at nag-a-analyze ng network traffic in real-time. Nmap — Network mapper na ginagamit para i-scan ang network at tukuyin ang mga open ports, services, at operating systems.

# Nmap: Scan for open ports
nmap -sV 192.168.1.1

# Nmap: Scan entire subnet
nmap -sS 192.168.1.0/24

# Wireshark TShark: Capture packets
tshark -i eth0 -c 100

Legal at Ethical Considerations

Ang ethical hacking ay ginagawa lamang may nakasulat na permission (authorization). Bawal mag-penetration test sa system na hindi mo pag-aari o walang permiso. Ang paggawa nito nang walang permiso ay labag sa batas (cybercrime). Laging sundin ang "rules of engagement" at i-report ang findings sa may-ari ng system.

"Ang ethical hacker ay 'good guy' — sinusubukan pasukin ang system para malaman ang butas at maayos bago pasukin ng totoong attacker."
