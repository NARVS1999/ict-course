Lesson 0042: Web Hosting & Deployment

Matuto kung paano i-deploy ang website para makita ng buong mundo — hosting providers, domains, FTP, at deployment platforms.

Lesson Goal Sa pagtatapos ng araling ito, ikaw ay makakapag-deploy ng static website sa GitHub Pages at makakapag-explain ng iba't ibang hosting options.

Ano ang Web Hosting?

Ang web hosting ay serbisyo na nag-i-store ng website files sa isang server na naka-konekta sa internet. Kapag nag-type ang user ng domain mo, ang hosting server ang magde-deliver ng files sa browser nila.

"Matapos mo gawin ang site, kailangan mong i-deploy para makita ng buong mundo. Maraming libreng hosting para sa practice."

Types of Hosting

  • Shared Hosting — Mura, maraming websites sa isang server. OK for small sites. (Hal. 000webhost, Hostinger)
  • VPS (Virtual Private Server) — Dedicated resources sa isang virtual server. Mas kontrolado.
  • Cloud Hosting — Scalable, binabayaran ang ginagamit. (Hal. AWS, Google Cloud, Azure)
  • Static Site Hosting — Libre, para sa HTML/CSS/JS sites. (Hal. GitHub Pages, Netlify, Vercel)

FTP at cPanel

Ang FTP (File Transfer Protocol) ay ginagamit para mag-upload ng files sa hosting server. Gamit ang cPanel, pwedeng ma-manage ang hosting — mag-upload ng files, gumawa ng databases, mag-set ng email, at mag-install ng CMS.

Gamit ang FileZilla (FTP client):
Host: ftp.yoursite.com
Username: yourusername
Password: ********
Port: 21

I-drag at i-drop ang files para ma-upload sa server.

Deploying Static Sites

GitHub Pages

1. Gumawa ng repository sa GitHub
2. I-push ang HTML files sa repository
3. Pumunta sa Settings → Pages
4. Piliin ang branch (main) at folder (/root)
5. I-access ang site sa:
   https://username.github.io/repository-name/

Netlify

1. Mag-sign up sa netlify.com
2. I-click ang "New site from Git"
3. I-connect ang GitHub repository
4. Awtomatikong made-deploy ang site
5. Pwedeng mag-custom domain

Domain Setup at DNS

Bumili ng domain name (hal. namecheap.com, GoDaddy). I-point ang DNS sa hosting provider sa pamamagitan ng pag-update ng nameservers o pag-add ng A record / CNAME record.

SSL Certificates

Ang SSL (Secure Sockets Layer) ay nag-e-encrypt ng data sa pagitan ng browser at server. Ang HTTPS ay naka-SSL na connection. Gamit ang Let's Encrypt, libre ang SSL certificate. Karamihan ng modern hosting (kasama ang GitHub Pages at Netlify) ay may automatic SSL.

Summary

  • Ang web hosting ay nag-iimbak ng website files sa isang server.
  • Ang GitHub Pages at Netlify ay libre para sa static sites.
  • Ang SSL certificate ay nag-e-encrypt ng data para sa secure na connection.
