Lesson 0054: Encryption & Data Protection

Pag-unawa sa encryption, hashing, SSL/TLS, at kung paano protektahan ang data sa rest at in transit.

Lesson Goal Matutunan ang pagkakaiba ng symmetric at asymmetric encryption, ang gamit ng hashing para sa passwords, at ang SSL/TLS certificates.

Symmetric vs Asymmetric Encryption

Sa symmetric encryption, iisang key ang ginagamit para mag-encrypt at mag-decrypt ng data. Mabilis ito pero may problema sa pag-share ng key. Sa asymmetric encryption, may dalawang keys: public key (pwedeng ibahagi) at private key (lihim). Gamit ang public key, kahit sino ay pwedeng mag-encrypt ng mensahe, ngunit ang private key lang ang makakapag-decrypt nito. Halimbawa: RSA, ECC.

Hashing para sa Passwords

Ang hashing ay isang one-way function — hindi pwedeng i-reverse para makuha ang original na input. Ginagamit ito para sa pag-imbak ng passwords: i-hash ang password at i-store ang hash. Kapag nag-login ang user, i-hash ang input at i-compare sa stored hash. Ang SHA-256 ay isang hash function, ngunit para sa passwords, mas mainam ang bcrypt o Argon2 dahil may salt at mas mabagal (resistant sa brute force).

<?php
// Magandang paraan: bcrypt
$password = "user_password123";
$hash = password_hash($password, PASSWORD_BCRYPT);

// I-verify kapag nag-login
if (password_verify($input_password, $stored_hash)) {
    echo "Login successful";
} else {
    echo "Invalid password";
}
?>

// Halimbawa ng SHA-256 (huwag gamitin for passwords alone)
$hash = hash('sha256', $password);
// Mas mainam ang bcrypt dahil may built-in salt at mas mabagal

SSL/TLS Certificates at HTTPS

Ang SSL/TLS ay protocols na nag-e-encrypt ng communication sa pagitan ng browser at server. Kapag ang website ay may HTTPS (SSL certificate), ang lahat ng data na ipinadala — kabilang ang passwords, credit card numbers — ay naka-encrypt at hindi nababasa ng mga nasa gitna (man-in-the-middle).

Full Disk Encryption

Ang full disk encryption ay nag-e-encrypt ng buong hard drive para kung manakaw o mawala ang computer, hindi mababasa ang data. Windows: BitLocker (kailangan ng TPM chip). macOS: FileVault. Linux: LUKS.

"Ang encryption ay parang sobre ng iyong data — tanging ang may tamang susi ang makakabasa ng laman nito."

Mini-Project: Check HTTPS Certificate

Pumunta sa isang website na gumagamit ng HTTPS (halimbawa: https://www.google.com). I-click ang padlock icon sa address bar para makita ang SSL certificate details. I-check ang: issuer, expiration date, at kung valid ang certificate. I-screenshot ang certificate details.

Summary

  • Symmetric = iisang key; Asymmetric = public/private key pair.
  • Hashing = one-way; bcrypt/Argon2 ang recommended para sa passwords.
  • HTTPS (SSL/TLS) ay nag-e-encrypt ng communication; full disk encryption ay nagproprotekta kung manakaw ang device.
