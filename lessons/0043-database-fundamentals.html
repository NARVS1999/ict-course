Lesson 0043: Database Fundamentals

Pag-unawa sa database, DBMS, relational vs NoSQL, at mga pangunahing konsepto tulad ng tables, records, at keys.

Lesson Goal Maunawaan kung ano ang database, ang mga uri nito, at ang mga pangunahing bahagi ng relational database.

Ano ang Database?

Ang database ay isang organisadong koleksyon ng data na madaling i-access, i-manage, at i-update. Halimbawa: listahan ng students sa isang school, inventory ng isang tindahan, o user accounts ng isang website.

DBMS (Database Management System)

Ang DBMS ay software na ginagamit para i-manage ang database. Mga halimbawa: MySQL (popular sa web), PostgreSQL (advanced features), at SQLite (lightweight, walang server). Pumili batay sa laki ng project at requirements.

Relational vs NoSQL

Ang relational database (tulad ng MySQL at PostgreSQL) ay naka-organize sa tables na may rows at columns, at may relasyon sa isa't isa gamit ang keys. Ang NoSQL (tulad ng MongoDB) naman ay flexible — pwedeng document-based, key-value, o graph — at hindi gumagamit ng fixed schema. Mainam ang relational para sa structured data at transactions; ang NoSQL ay para sa malalaking volume ng unstructured data.

"Ang relational database naka-organize sa tables na parang mga spreadsheet na may relasyon sa isa't isa."

Tables, Records, at Fields

Ang table ay binubuo ng rows (records) at columns (fields). Halimbawa, sa isang "Students" table, ang bawat row ay isang estudyante, at ang columns ay: student_id, first_name, last_name, at email.

Primary Key at Foreign Key

Ang Primary Key ay isang unique identifier para sa bawat record sa isang table — hindi pwedeng magkaroon ng dalawang records na pareho ang primary key. Ang Foreign Key naman ay isang column na tumutukoy sa primary key ng ibang table, na ginagamit para i-link ang dalawang tables.

-- Halimbawa ng paggawa ng table na may primary key
CREATE TABLE Students (
    student_id INT PRIMARY KEY,
    first_name VARCHAR(50),
    last_name VARCHAR(50),
    email VARCHAR(100)
);

-- Halimbawa ng foreign key
CREATE TABLE Enrollments (
    enrollment_id INT PRIMARY KEY,
    student_id INT,
    course_code VARCHAR(10),
    FOREIGN KEY (student_id) REFERENCES Students(student_id)
);

Summary

  • Ang database ay organisadong koleksyon ng data na mina-manage ng DBMS.
  • Relational databases gumagamit ng tables na may primary at foreign keys; NoSQL ay flexible schema.
  • Ang primary key ay unique identifier; foreign key ay nagli-link ng tables.
