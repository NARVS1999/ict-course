Lesson 0053: Web Application Security (OWASP Top 10)

Pag-unawa sa mga pinakakaraniwang vulnerabilities sa web apps: SQL injection, XSS, CSRF, at iba pa.

Lesson Goal Matutunan ang OWASP Top 10 vulnerabilities at kung paano maiiwasan ang mga ito sa iyong web applications.

SQL Injection

Ang SQL Injection ay nangyayari kapag ang attacker ay nagpasok ng malicious SQL code sa pamamagitan ng user input. Prevention: gumamit ng prepared statements (parameterized queries), huwag mag-concatenate ng user input, at i-validate ang input.

// VULNERABLE: Huwag gayahin ito
$query = "SELECT * FROM users WHERE username = '" . $_POST['username'] . "'";
$result = mysqli_query($conn, $query);

// SECURE: Gamitin ang prepared statements
$stmt = $pdo->prepare("SELECT * FROM users WHERE username = :username");
$stmt->execute([':username' => $_POST['username']]);

Cross-Site Scripting (XSS)

Ang XSS ay kapag ang attacker ay nag-inject ng malicious JavaScript sa web page na tinitignan ng ibang users. Prevention: i-escape (sanitize) ang output bago i-display. Sa PHP, gamitin ang htmlspecialchars(). Huwag magtiwala sa user input kahit saan.

// VULNERABLE: Diretso display ng user input
echo "<h1>Welcome, " . $_GET['name'] . "</h1>";

// SECURE: I-escape ang output
echo "<h1>Welcome, " . htmlspecialchars($_GET['name'], ENT_QUOTES, 'UTF-8') . "</h1>";

Cross-Site Request Forgery (CSRF)

Ang CSRF ay pinipilit ang user na mag-execute ng unwanted action (halimbawa: mag-change ng password) sa isang website kung saan sila ay naka-login. Prevention: gumamit ng CSRF token — isang unique, random na value na naka-embed sa forms at na-verify ng server.

<?php
session_start();
// Generate CSRF token
if (empty($_SESSION['csrf_token'])) {
    $_SESSION['csrf_token'] = bin2hex(random_bytes(32));
}
?>
<form method="POST">
    <input type="hidden" name="csrf_token" value="<?php echo $_SESSION['csrf_token']; ?>">
    <input type="submit" value="Delete Account">
</form>

<?php
// Verify CSRF token sa server
if ($_POST['csrf_token'] !== $_SESSION['csrf_token']) {
    die('CSRF token validation failed');
}
?>

Iba Pang OWASP Top 10

Sensitive Data Exposure — Huwag mag-imbak ng passwords sa plain text; gumamit ng encryption para sa sensitive data. Security Misconfiguration — I-disable ang default accounts, i-remove ang unnecessary features, panatilihing updated ang software. Broken Access Control — I-verify na ang user ay may permission bago i-access ang resources.

Summary

  • SQL Injection = malisiyosong SQL sa input — iwasan gamit ang prepared statements.
  • XSS = malisiyosong JavaScript — i-prevent sa pamamagitan ng output escaping.
  • CSRF = unwanted actions — i-prevent gamit ang CSRF tokens.
