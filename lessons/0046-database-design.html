Lesson 0046: Database Design & Normalization

Pagdidisenyo ng maayos na database gamit ang ERD at normalization para maiwasan ang redundancy.

Lesson Goal Matutunan ang Entity-Relationship Diagrams, normalization (1NF, 2NF, 3NF), at kung paano mag-disenyo ng efficient na schema.

Entity-Relationship Diagram (ERD)

Ang ERD ay isang visual na representasyon ng mga entities (tables) at ang relasyon nila sa isa't isa. Halimbawa, ang "Student" ay may relasyon na "enrolls in" sa "Course." Ginagamit ang ERD para magplano ng database bago ito gawin.

Normalization: 1NF

Ang First Normal Form (1NF) ay nangangailangan na ang bawat column ay naglalaman ng iisang value (atomic), at ang bawat row ay may unique identifier (primary key). Iwasan ang paglalagay ng maraming values sa isang column gaya ng "Math, Science, English."

Normalization: 2NF

Ang Second Normal Form (2NF) ay dapat nasa 1NF na, at ang bawat non-key column ay dapat fully dependent sa buong primary key — hindi lang sa bahagi nito. Ito ay importante kapag gumagamit ng composite primary key.

Normalization: 3NF

Ang Third Normal Form (3NF) ay dapat nasa 2NF na, at walang transitive dependency — ibig sabihin, ang non-key columns ay hindi dapat dependent sa ibang non-key columns. Halimbawa, kung may "CourseCode" ka, huwag ilagay ang "Department" sa parehong table kung ang "Department" ay depende sa "CourseCode" at hindi sa primary key.

"Huwag maglagay ng paulit-ulit na data — i-normalize para iwas redundancy at inconsistency."

Denormalization Trade-offs

Minsan, sinasadya ng designers na i-denormalize (magdagdag ng redundancy) para pabilisin ang read operations. Halimbawa, sa isang reporting database, pwedeng mag-imbak ng "total_sales" sa isang table kahit kalkulado ito mula sa ibang table. Trade-off: mas mabilis magbasa, pero mas mahirap i-maintain ang consistency.

Pag-disenyo ng Schema

Bago gumawa ng database, planuhin ang schema: tukuyin ang entities, attributes, at relasyon. Siguraduhing naka-normalize hanggang 3NF para iwas data anomalies.

-- Halimbawa ng normalized schema para sa Library System
CREATE TABLE Members (
    member_id INT PRIMARY KEY,
    name VARCHAR(100),
    email VARCHAR(100)
);

CREATE TABLE Books (
    book_id INT PRIMARY KEY,
    title VARCHAR(200),
    author VARCHAR(100),
    isbn VARCHAR(20)
);

CREATE TABLE Loans (
    loan_id INT PRIMARY KEY,
    member_id INT,
    book_id INT,
    loan_date DATE,
    return_date DATE,
    FOREIGN KEY (member_id) REFERENCES Members(member_id),
    FOREIGN KEY (book_id) REFERENCES Books(book_id)
);

Summary

  • Ang ERD ay visual na plano ng database bago ito gawin.
  • 1NF = atomic columns at unique rows; 2NF = full dependency sa primary key; 3NF = walang transitive dependency.
  • Normalization = iwas redundancy; denormalization = pabilisin ang reads (trade-off).
