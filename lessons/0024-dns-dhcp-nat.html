Lesson 0024: DNS, DHCP & NAT

Alamin kung paano gumagana ang DNS, DHCP, at NAT — tatlong mahahalagang serbisyo sa networking.

Lesson Goal Maunawaan ang role ng DNS sa pag-resolve ng domain names, DHCP sa automatic IP assignment, at NAT sa pag-share ng public IP.

DNS (Domain Name System)

Ang DNS ay parang phonebook ng internet — nagco-convert ng domain names (hal. google.com) sa IP addresses (142.250.80.46). Kapag nag-type ka ng website address sa browser, nagtatanong ang computer mo sa DNS server para malaman ang IP. May iba't ibang DNS records: A record (IPv4 address), CNAME (alias ng domain), MX (mail exchange para sa email). Kung walang DNS, kailangan nating i-memorize ang IP addresses ng bawat website — impossible!

DHCP (Dynamic Host Configuration Protocol)

Ang DHCP ay awtomatikong nag-a-assign ng IP address, subnet mask, default gateway, at DNS server sa mga devices sa network. Kapag nag-connect ka sa Wi-Fi, ang router (na may DHCP server) ang nagbibigay ng IP mo. Ang process ay tinatawag na DORA: Discover (hanap ang DHCP server), Offer (offer ng IP), Request (request ng IP), Acknowledge (confirm). Ang lease ay ang tagal ng paggamit ng IP — magre-renew ito bago mag-expire.

NAT (Network Address Translation)

Ang NAT ay nagbibigay-daan sa maraming devices na mag-share ng iisang public IP address. Kapag pumunta ka sa internet, ang router mo ay nagta-translate ng private IP mo (192.168.1.10) sa public IP na binigay ng ISP. Ang lahat ng devices sa bahay ay lumalabas na iisang public IP lang sa internet. Ang port forwarding naman ay nag-aallow ng external access sa specific device sa loob ng network — halimbawa, para sa gaming server o CCTV.

"Kapag nag-connect ka sa Wi-Fi, si DHCP magbibigay sa'yo ng IP. Si DNS ang maghahanap ng address ng website."

Mini-Project: Use nslookup

Buksan ang Command Prompt (cmd) at i-type ang nslookup google.com. Tingnan ang IP addresses na lumabas — ito ang mga Google servers. Subukan din ang nslookup facebook.com at nslookup google.com 8.8.8.8 (gamit ang Google DNS). I-compare ang results. Isulat ang mga IP addresses na nakuha mo.

Summary

  • DNS ang nagco-convert ng domain names sa IP addresses — parang phonebook ng internet.
  • DHCP ang awtomatikong nag-a-assign ng IP, subnet mask, at gateway sa mga devices.
  • NAT ang nagpapahintulot na mag-share ng iisang public IP ang maraming devices.
  • Port forwarding ay nagbubukas ng specific ports para sa external access sa local devices.
