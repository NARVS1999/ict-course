Lesson 0033: Version Control with Git & GitHub

Matuto ng Git para i-track ang changes ng code at gamitin ang GitHub para sa collaboration.

Lesson Goal Sa pagtatapos ng araling ito, ikaw ay makakagawa ng Git repository, makakapag-commit ng changes, at makakapag-push sa GitHub.

Git Concepts

Ang Git ay version control system na nagta-track ng bawat pagbabago sa code. Repository ang tawag sa project folder na may Git. Ang commit ay snapshot ng files sa isang specific na time. Ang branch ay hiwalay na version ng code para magtrabaho nang hindi naaapektuhan ang main code.

"Ang Git parang time machine — kung nagkamali ka, kaya mong bumalik sa nakaraang version ng code."

Basic Git Commands

# I-setup ang Git sa folder
git init                    # Gumawa ng bagong repository
git status                  # Tingnan ang status ng files

# Mag-stage at mag-commit
git add index.html          # I-stage ang file para i-commit
git add .                   # I-stage lahat ng files
git commit -m "Initial commit"  # Mag-save ng snapshot

# Mag-branch
git branch feature-login    # Gumawa ng bagong branch
git checkout feature-login  # Lumipat sa branch
git merge feature-login     # Pagsamahin ang branch sa main

# Mag-connect sa remote
git remote add origin https://github.com/user/repo.git
git push origin main        # I-push ang commits sa GitHub
git pull origin main        # I-download ang latest changes

GitHub Collaboration

Ang GitHub ay online platform para mag-store ng Git repositories. Pwede kang mag-upload ng code, mag-collaborate sa iba, at mag-publish ng projects. Ang pull request ay request para i-merge ang changes mula sa isang branch papunta sa main. Ang README.md ay parang front page ng repository na naglalaman ng description at instructions.

Mini-Project: First GitHub Repo

Gumawa ng GitHub account kung wala pa. I-create ang repository na "my-first-repo". I-initialize ito ng README. I-clone sa local machine, mag-add ng index.html file, i-commit at i-push pabalik sa GitHub.

Summary

  • Ang Git ay version control system para i-track ang code changes.
  • Ang git init, add, commit, push ay essential commands.
  • Ang GitHub ay online platform para sa collaboration at pag-store ng repositories.
