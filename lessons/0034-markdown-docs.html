Lesson 0034: Markdown & Documentation

Matuto ng Markdown syntax para gumawa ng malinis at professional na documentation.

Lesson Goal Sa pagtatapos ng araling ito, ikaw ay makakagawa ng README.md at makakapagsulat ng Markdown documents na may headings, lists, links, images, at code blocks.

Markdown Syntax

Ang Markdown ay lightweight markup language para mag-format ng text. Ginagamit ito sa README files, documentation, forums, at note-taking apps.

"Lahat ng ICT professional marunong mag-Markdown — simple lang pero sobrang linis ng documentation."
# Heading 1 (Tulad ng h1 sa HTML)
## Heading 2
### Heading 3

**Bold text** at *italic text*

- Unordered list item
- Another item
  - Nested item

1. Ordered list item
2. Second item

[Link text](https://example.com)

![Alt text](image.jpg)

`Inline code`

```python
# Code block
print("Hello, Markdown!")
```

README Files

Ang README.md ang unang nakikita ng mga tao sa isang GitHub repository. Dapat ito naglalaman ng:

  • Project title at description
  • Installation instructions
  • Usage examples
  • License at contact info

Documentation Tools

Para sa mas malaking documentation projects, gamitin ang MkDocs (Python-based) o Docusaurus (React-based). Pareho silang gumagamit ng Markdown files at nagge-generate ng static HTML site.

Mini-Project: Project README

Gumawa ng README.md para sa isang sample project na "Student Grade Calculator". Isama ang: title, description, installation steps, usage example, at license. Gamitin ang Markdown syntax na natutunan.

Summary

  • Ang Markdown ay lightweight markup para sa text formatting.
  • Gamitin ang # para sa headings, ** para sa bold, at ``` para sa code blocks.
  • Ang README.md ay mahalaga sa bawat project para sa documentation.
