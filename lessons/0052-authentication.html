Lesson 0052: Authentication & Access Control

Pag-unawa sa MFA, password policies, least privilege, at role-based access control.

Lesson Goal Matutunan kung paano patatagin ang authentication at i-manage ang access ng users sa system.

Multi-Factor Authentication (MFA)

Ang MFA ay gumagamit ng dalawa o higit pang paraan para i-verify ang identity ng user: something you know (password), something you have (phone/authenticator app), at something you are (fingerprint/face). Kahit makompromiso ang password, hindi makakapasok ang attacker kung wala ang second factor.

Password Policies

Ang magandang password policy ay: gumamit ng mahaba at complex na password (minimum 12 characters, may uppercase, lowercase, numbers, symbols), huwag mag-reuse ng passwords sa iba't ibang accounts, at palitan ang password kung may suspected breach. Gamitin ang password manager (LastPass, Bitwarden, 1Password) para hindi mo kailanganing tandaan lahat.

Principle of Least Privilege

Ang bawat user o system ay dapat bigyan lamang ng pinakamababang permissions na kailangan para magawa ang trabaho. Halimbawa, ang isang intern ay hindi dapat magkaroon ng admin access. Kung ma-compromise ang account, limitado lamang ang pwedeng gawin ng attacker.

RBAC (Role-Based Access Control)

Sa RBAC, ang permissions ay naka-assign sa roles, hindi sa individual users. Halimbawa: "Admin" role ay may full access, "Editor" ay pwedeng mag-edit ng content, "Viewer" ay pwedeng tumingin lamang. Kapag nagpalit ng role ang user, nagbabago rin ang access niya.

"Gumamit ng password manager at i-enable ang 2-factor authentication sa lahat ng important accounts."
-- Halimbawa ng RBAC database schema
CREATE TABLE Roles (
    role_id INT PRIMARY KEY,
    role_name VARCHAR(50)
);

CREATE TABLE Permissions (
    perm_id INT PRIMARY KEY,
    perm_name VARCHAR(100)
);

CREATE TABLE Role_Permissions (
    role_id INT,
    perm_id INT,
    PRIMARY KEY (role_id, perm_id),
    FOREIGN KEY (role_id) REFERENCES Roles(role_id),
    FOREIGN KEY (perm_id) REFERENCES Permissions(perm_id)
);

CREATE TABLE User_Roles (
    user_id INT,
    role_id INT,
    PRIMARY KEY (user_id, role_id),
    FOREIGN KEY (role_id) REFERENCES Roles(role_id)
);

Mini-Project: Enable 2FA

Pumili ng isang account (Google, Facebook, GitHub, o iba pa) at i-enable ang 2-Factor Authentication. I-document ang steps: pumunta sa security settings, piliin ang authentication method (authenticator app o SMS), at i-verify na gumagana ito. Huwag ibahagi ang iyong screenshots na may sensitive info.

Summary

  • MFA = maraming layers ng verification; hindi sapat ang password lang.
  • Least privilege = minimal access lang; password manager = secure storage.
  • RBAC = permissions ay naka-assign sa roles, mas madali i-manage.
