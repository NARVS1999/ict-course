Lesson 0065: Capstone Presentation & Review

Paghahanda para sa huling presentasyon ng iyong capstone project — demo, peer review, at kung paano magpatuloy sa iyong ICT journey.

Lesson Goal Sa pagtatapos ng araling ito, ikaw ay makapaghahanda ng 5-minute project presentation, makakapagbigay at makakatanggap ng peer feedback, at makakagawa ng plano para sa patuloy na pag-aaral.

Pagpresenta ng Iyong Project

Ang presentasyon ay ang pagkakataon mo na i-showcase ang iyong natutunan at ang iyong project. Narito ang structure ng isang magandang project demo:

  • Introduction (30 sec) — Sino ka at ano ang project mo?
  • Problem (30 sec) — Anong problema ang ni-solve ng project mo?
  • Demo (2 min) — Ipakita ang iyong project. Mag-start sa homepage, i-demo ang main features.
  • Tech Stack (1 min) — Anong technologies ang ginamit mo? Bakit mo sila pinili?
  • Challenges & Learnings (30 sec) — Anong challenges ang na-encounter mo at paano mo ito na-solve?
  • Conclusion & Next Steps (30 sec) — Ano ang next plans para sa project? Anong gusto mong matutunan next?
"Natapos mo man ang guide na ito, tandaan: sa ICT, tuloy-tuloy ang pag-aaral. Laging may bago. Build, break, fix, learn. Good luck sa journey mo!"

Peer Review

Ang peer review ay ang proseso kung saan sinusuri ng mga kaklase o kasamahan ang iyong project at nagbibigay ng feedback. Mahalaga ito dahil:

  • Nakakakuha ka ng ibang perspective sa project mo.
  • Nakakakita ng potential improvements na hindi mo napansin.
  • Natututo ka sa projects ng iba.

Kapag nagbibigay ng feedback: maging constructive at specific. "Maganda ang UI mo, pero baka pwedeng magdagdag ng search function" — mas helpful ito kaysa "maganda" lang.

Next Steps: Continuous Learning

Ang pagkakatapos ng kursong ito ay hindi ang katapusan — simula pa lang ito. Narito ang mga susunod mong pwedeng gawin:

  • Magpatuloy sa pag-gawa ng personal projects.
  • Sumali sa tech communities (Facebook groups, Discord, Reddit).
  • Kumuha ng IT certifications (CompTIA A+, AWS Cloud Practitioner, atbp.)
  • Mag-internship o mag-freelance sa platforms tulad ng Upwork o OnlineJobs.ph.
  • Mag-explore ng advanced topics: cybersecurity, AI/ML, cloud architecture.

Quick Quiz

1. Ano ang dapat isama sa isang project demo?
2. Bakit mahalaga ang peer review?
3. Ano ang ibig sabihin ng continuous learning sa ICT?

Mini-Project: Prepare Your Capstone Presentation

Gumawa ng 5-minute presentation para sa iyong capstone project. Maaaring gumamit ng PowerPoint, Google Slides, o kahit ano pang presentation tool. Siguraduhing may slides para sa: Title, Problem, Solution/Demo, Tech Stack, Challenges, at Next Steps. I-present ito sa harap ng klase o i-record ang presentation video.

Summary

  • Ang isang magandang project demo ay may introduction, problem, demo, tech stack, challenges, at next steps.
  • Ang peer review ay nagbibigay ng valuable feedback at ibang perspective.
  • Ang continuous learning ay susi sa ICT — laging may bagong technologies na dapat pag-aralan.
