Lesson 0025: Basic Network Troubleshooting Commands

Matutunan ang mga essential command-line tools para i-diagnose ang network problems — ping, tracert, ipconfig, nslookup, netstat, at arp.

Lesson Goal Magamit ang basic network troubleshooting commands para i-check ang connectivity, i-diagnose ang issues, at maunawaan ang network configuration.

Ping — Test Connectivity

Ang ping ay nagse-send ng ICMP echo request packets sa isang destination IP o domain name at sumusukat kung gaano katagal bago bumalik ang reply. Gamitin ito para i-check kung may connection sa isang device o website. Halimbawa: ping google.com o ping 8.8.8.8. Kung may reply, ibig sabihin may connection. Kung "Request timed out", walang connection o naka-block ang ICMP.

"Kapag walang internet, unang gawin: mag-ping sa 8.8.8.8 (Google DNS). Kapag may reply, ibig sabihin may connection pero baka DNS issue."

Tracert/Traceroute — Path ng Packets

Ang tracert (Windows) o traceroute (macOS/Linux) ay nagpapakita ng path na tinatahak ng packets papunta sa destination. Makikita mo ang bawat hop (router) na dinadaanan at kung gaano katagal. Gamitin ito para malaman kung saan nagka-problema ang connection — hal. tracert google.com.

Ipconfig/Ifconfig — IP Configuration

Ang ipconfig (Windows) ay nagpapakita ng IP address, subnet mask, default gateway, at DNS servers ng iyong computer. Gamitin ang ipconfig /all para sa detailed info (MAC address, DHCP status). Ang ipconfig /release at ipconfig /renew ay nagre-release at nagre-renew ng IP lease.

Nslookup, Netstat, at ARP

nslookup — query sa DNS para malaman ang IP address ng domain name. netstat — nagpapakita ng active connections, listening ports, at network statistics. Gamitin ang netstat -an para sa lahat ng connections at ports. arp -a — nagpapakita ng ARP table (IP-to-MAC address mappings) ng mga devices na nakausap ng computer mo.

Quick Quiz

1. Ano ang ginagawa ng ping command?
2. Ano ang ipinapakita ng ipconfig?
3. Ano ang 8.8.8.8?

Mini-Project: Run Network Commands

Buksan ang Command Prompt at isa-isang gawin ang mga sumusunod na commands: ipconfig /all, ping google.com, tracert google.com, nslookup google.com, at arp -a. I-screenshot o kopyahin ang output ng bawat command sa isang document. Isulat sa tabi kung ano ang ibig sabihin ng resulta — hal. "Ang IP ko ay 192.168.1.10, ang ping sa Google ay 15ms."

Summary

  • Ping para i-test kung may connectivity sa destination — gamitin ang 8.8.8.8 para i-check ang internet.
  • Tracert para makita ang path ng packets at malaman kung saan nagka-problema.
  • Ipconfig para makita ang IP configuration ng computer.
  • Nslookup para i-query ang DNS; netstat para sa active connections; arp para sa IP-to-MAC mappings.
