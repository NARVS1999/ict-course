Networking Commands Cheatsheet

Mga command para i-diagnose ang network problems — dapat kabisado!

Windows Commands

CommandPurpose
ipconfigIpakita ang IP address, subnet mask, default gateway
ipconfig /allDetailed network info (MAC address, DNS, DHCP)
ping <address>Mag-test ng connectivity sa device/server
tracert <address>I-trace ang route papunta sa destination
nslookup <domain>I-resolve ang domain name sa IP address
netstat -anIpakita ang active connections at ports
arp -aIpakita ang IP-to-MAC address table
ipconfig /flushdnsI-clear ang DNS cache
ipconfig /release && ipconfig /renewMag-release at mag-renew ng IP lease

Linux / macOS Commands

CommandPurpose
ifconfigIpakita ang network interfaces at IP
ip aModern alternative sa ifconfig
ping <address>Test connectivity
traceroute <address>I-trace ang route
nslookup <domain>DNS lookup
dig <domain>Detailed DNS lookup
netstat -tulnIpakita ang listening ports
ss -tulnModern alternative sa netstat
arp -aARP cache

Troubleshooting Flow

1
ping 8.8.8.8 — may internet connection ba? Kung may reply, may connection.
2
ping google.com — gumagana ba ang DNS? Kung nag-resolve, DNS ok.
3
ipconfig/ifconfig — may IP address ka ba? Check kung naka-DHCP.
4
tracert/traceroute 8.8.8.8 — saan banda humihinto ang connection?
5
nslookup google.com — nagre-resolve ba ang DNS?
