Git & GitHub Cheatsheet

Ang mga pinakagamit na Git commands — parang time machine para sa code mo!

Setting Up

git config --global user.name "Name"
git config --global user.email "email@example.com"
git init            — mag-start ng bagong repository
git clone <url>    — kopyahin ang existing repo

Basic Workflow

git status          — tingnan ang status ng files
git add <file>     — i-stage ang file para i-commit
git add .           — i-stage lahat ng files
git commit -m "msg" — i-save ang changes (may message)
git log --oneline   — tingnan ang commit history

Working with Remote (GitHub)

git remote add origin <url>   — i-link ang local repo sa GitHub
git push -u origin main       — i-upload ang commits sa GitHub
git pull                      — i-download ang latest changes
git fetch                     — tingnan ang changes nang hindi merge

Branches

git branch                    — listahan ng branches
git branch <name>            — gumawa ng bagong branch
git checkout <branch>        — lumipat ng branch
git checkout -b <name>       — gumawa at lumipat nang sabay
git merge <branch>           — i-merge ang branch sa current branch

Undoing Changes

git restore <file>           — i-undo ang unstaged changes
git restore --staged <file>  — i-unstage ang file
git reset --soft HEAD~1      — i-undo ang last commit (keep changes)
git reset --hard HEAD~1      — i-undo ang last commit (delete changes)

GitHub Pull Request Flow

1. git checkout -b feature-branch
2. (gumawa ng changes, git add, git commit)
3. git push -u origin feature-branch
4. Sa GitHub: Create Pull Request
5. (pagkatapos ma-approve, i-merge)
6. git checkout main
7. git pull
